Privacy Policy

Last Updated: March 2026

1. Technical Overview

NoRelapse is a discipline-building tool designed with a "Safety-First, Data-Last" architecture. We prioritize on-device processing to ensure maximum user privacy.

2. Accessibility Service Usage

NoRelapse utilizes the AccessibilityService API to enable its core "Guardian Shield" protocol. This service is mandatory for identifying and intercepting browser-based triggers.

  • URL Monitoring: Real-time inspection of browser addresses against local blocklists.
  • Keyword Filtering: Identification of relapse-triggering search terms inside browser fields.
  • Zero Data Transmission: All monitored data is processed locally. We do not store, upload, or share your browsing history.

3. Request Install Packages

We utilize the Request Install Packages permission solely to facilitate secure self-updates. This ensures that the application can receive critical security patches and "Guardian Shield" optimization directly from our servers.

4. Data Collection

NoRelapse does not collect personal identifiers, location data, or contact lists. Any data stored within the app (such as streaks and local settings) remains encrypted on your device.
